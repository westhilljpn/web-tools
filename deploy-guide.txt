================================================================
  Vercel デプロイ手順（非エンジニア向け・PC操作ベース）
================================================================

【全体の流れ】
  GitHub にコードを置く → Vercel と連携する → 自動でサイトが公開される


----------------------------------------------------------------
STEP 1: GitHub にアカウントを作る（持っていない場合）
----------------------------------------------------------------
1. ブラウザで https://github.com を開く
2. 右上の「Sign up」をクリック
3. メールアドレス・パスワード・ユーザー名を入力して登録


----------------------------------------------------------------
STEP 2: GitHub にリポジトリを作る
----------------------------------------------------------------
1. ログイン後、右上の「+」→「New repository」をクリック
2. 以下を入力：
     Repository name : web-tools（好きな名前でOK）
     Public / Private: どちらでもOK（Private でも無料）
3. 「Create repository」をクリック


----------------------------------------------------------------
STEP 3: ローカルのコードを GitHub に送る
----------------------------------------------------------------
ターミナルで以下を順番に実行
（「あなたのGitHubユーザー名」は実際の名前に置き換える）

  cd /home/taki/projects/web-tools
  git remote add origin https://github.com/あなたのGitHubユーザー名/web-tools.git
  git push -u origin main

→ GitHub のユーザー名・パスワード（またはトークン）を聞かれたら入力する
→ ブラウザで https://github.com/あなたのGitHubユーザー名/web-tools を開いて
  コードが表示されていれば OK

  ★ パスワードを求められてエラーになる場合
     GitHub は 2021 年以降パスワード認証を廃止しています。
     「Personal Access Token（PAT）」が必要です。
     GitHub の Settings → Developer settings → Personal access tokens
     から発行してください（repo 権限にチェック）。
     発行したトークンをパスワードの代わりに入力します。


----------------------------------------------------------------
STEP 4: Vercel にアカウントを作る
----------------------------------------------------------------
1. ブラウザで https://vercel.com を開く
2. 「Sign Up」をクリック
3. 「Continue with GitHub」を選ぶ
4. GitHub の認証画面が出たら「Authorize Vercel」をクリック


----------------------------------------------------------------
STEP 5: Vercel にプロジェクトをインポートする
----------------------------------------------------------------
1. Vercel のダッシュボードで「Add New... → Project」をクリック
2. GitHub のリポジトリ一覧が表示されるので、
   web-tools の右にある「Import」をクリック
3. 設定画面が開く：
     Framework Preset : Next.js が自動で選ばれるはず（そのままでOK）
     Root Directory   : そのまま（変更不要）
     Environment Variables は次の STEP 6 で設定


----------------------------------------------------------------
STEP 6: 環境変数を設定する
----------------------------------------------------------------
STEP 5 の設定画面の下部「Environment Variables」欄に以下を追加：

  Key                          Value
  ---------------------------  -----------------------------------------
  NEXT_PUBLIC_SITE_URL         https://あなたのプロジェクト名.vercel.app
  NEXT_PUBLIC_ADSENSE_ENABLED  false

※ NEXT_PUBLIC_SITE_URL の値はこの時点で正確にわからなくてOK。
  デプロイ後に URL が確定したら、Vercel の Settings から後で修正できます。


----------------------------------------------------------------
STEP 7: デプロイ実行
----------------------------------------------------------------
設定が終わったら「Deploy」ボタンをクリック。

→ 2〜3 分ほどでビルドが完了し、
  https://web-tools-xxxxx.vercel.app のような URL でサイトが公開されます
→ デプロイ完了後、表示された URL をブラウザで開いて確認する


----------------------------------------------------------------
STEP 8: 本番 URL を README.md に記録する
----------------------------------------------------------------
確定した URL を README.md の「本番 URL」行に書き込む：

  **本番 URL**: https://web-tools-xxxxx.vercel.app


----------------------------------------------------------------
今後の運用（コードを更新したとき）
----------------------------------------------------------------
コードを変更して GitHub に git push するだけで、
Vercel が自動的に再デプロイしてくれます。手動操作は不要です。

  コード修正 → git commit → git push → 自動でサイト更新


----------------------------------------------------------------
つまずきやすいポイント まとめ
----------------------------------------------------------------
1. git push でパスワードエラー
   → Personal Access Token（PAT）を使う（STEP 3 参照）

2. 環境変数の URL が間違っている
   → サイトマップや canonical URL がズレますが、
     後から Vercel の Settings 画面でいつでも修正できます

3. ビルドエラーで Deploy が失敗する
   → Vercel のダッシュボードでエラーログを確認し、
     ローカルで npm run build を実行して原因を特定する
