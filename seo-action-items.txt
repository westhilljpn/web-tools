■ 手動対応が必要な事項
================================================================

------------------------------------------------------------
🔴 優先度1（今すぐ）
------------------------------------------------------------

1. GSC：サイトマップを再送信
   手順: Google Search Console
         → サイドメニュー「サイトマップ」
         → https://quicker-app.com/sitemap.xml を入力して「送信」
   理由: robots.txt のサイトマップURLを修正したため、
         再送信してGoogleに認識させる必要がある。

2. GSC：インデックス申請（「検出・インデックス未登録」31件）
   手順: Google Search Console
         → 「URL検査」でURLを入力
         → 「インデックス登録をリクエスト」をクリック
         → 優先度の高いツール5〜10件から順に実施
   理由: Googleにサイトの存在は知られているが
         クロールされていないページが多数ある。


------------------------------------------------------------
🟠 優先度2（今週中）
------------------------------------------------------------

3. favicon を作成・設置
   手順: 512×512px の PNG を用意し
         src/app/icon.png（または public/favicon.ico）に配置
   理由: 現在 favicon が存在しない。
         検索結果・ブラウザタブでのCTRに直接影響する。

4. PageSpeed Insights でスコア確認
   手順: https://pagespeed.web.dev/ で
         https://quicker-app.com/ja/text-counter などを入力
         モバイル・デスクトップのスコアを記録する
   理由: Core Web Vitals (LCP/CLS/FID) の現状を把握してから
         優先して改善する箇所を決める。

5. Google Fonts を next/font に移行
   手順: src/app/[locale]/layout.tsx の <link rel="stylesheet"> を
         next/font/google（Noto Sans JP）に置き換える
   理由: 現在のやり方はレンダリングブロッキングになりやすく
         LCP スコアを下げる原因になる可能性がある。
         （コード変更が必要）


------------------------------------------------------------
🟡 優先度3（2週間以内）
------------------------------------------------------------

6. GSC データ確認 → CTR改善
   手順: Google Search Console → 「検索パフォーマンス」
         「表示回数」多い順で並べ、CTRが低いページを特定
         → title / meta description を書き直す
   理由: 既に検索に出ているが選ばれていないページが
         改善の伸びしろが最も大きい。

7. X（Twitter）で新ツール追加を告知
   手順: ツールを追加したタイミングで
         「○○ツールを追加しました: URL」の投稿を習慣化
   理由: ソーシャルシグナルの蓄積＋初期バックリンク獲得。

8. Zenn または Qiita に記事投稿
   例: 「自分で作った無料Webツール52選」
       「○○作業を効率化するブラウザツールを作った話」
   理由: 権威あるドメインからのバックリンクでドメイン評価を向上。
         ツールへの自然なリンクを含める形にする。


------------------------------------------------------------
🟢 優先度4（継続対応）
------------------------------------------------------------

9. はてなブックマークに登録
   理由: 便利ツール系コンテンツはブクマされやすい。
         はてな砲が来ると一時的に大量流入するケースもある。

10. カテゴリ別ランディングページの作成
    例: /ja/tools/text, /ja/tools/calculate など
    理由: 「テキストツール 無料」「計算ツール まとめ」などの
          カテゴリクエリで上位表示できる可能性がある。
          （コード変更が必要）

11. Web App Manifest の追加
    ファイル: src/app/manifest.ts
    理由: PWA対応・「ホームに追加」でブランド接触機会を増やす。
          （コード変更が必要）

12. GA で離脱率・平均滞在時間を確認
    手順: Google Analytics で各ツールページの
          平均エンゲージメント時間・離脱率を確認
    理由: 実際によく使われているツールを特定し、
          おすすめ表示の選定や重点改善対象の絞り込みに使う。
